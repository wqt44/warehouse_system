# 仓库系统：逻辑与规则清单

本文档列出当前仓库系统（warehouse_system）中的**所有逻辑与规则**，便于查阅与核对。

---

## 一、网格与地图

| 规则 | 说明 |
|------|------|
| **网格类型** | 0=空地，1=障碍物，2=货架，3=工作站，4=充电站 |
| **货架** | 由 `shelf_regions` 矩形区域填充为 2，边界会裁剪到网格内 |
| **工作站** | 由 `workstation_positions` 逐个设为 3 |
| **障碍物** | 由 `obstacle_positions` 设为 1，不覆盖已有 2/3/4 |
| **充电站** | 若配置了 `charging_stations` 则按位置设 4（且该格不能是 1 或 2）；否则在工作站相邻空地自动生成；若仍无则在网格中找第一个空地设 4 |

---

## 二、机器人初始化

| 规则 | 说明 |
|------|------|
| **出生格** | 仅允许在格子类型为 0（空地）、3（工作站）、4（充电站）的位置出生 |
| **不重复** | 每个机器人占一格，所有初始位置不重复 |
| **数量** | 机器人数量 = min(配置的 num_robots, 可用格数)；若无可用格则 fallback 为 (0,0) |

---

## 三、动作与移动（环境执行）

- **动作空间**：0=上，1=右，2=下，3=左，4=等待。

| 规则 | 说明 |
|------|------|
| **充电中不移动** | 若机器人在充电站且状态为 CHARGING，本步强制保持原位置，不响应移动动作 |
| **边界** | 越界则保持原位置 |
| **障碍(1)** | 目标格为障碍则保持原位置 |
| **货架(2)** | 仅当“有当前任务 + 状态为 MOVING_TO_PICKUP + 目标格为当前任务取货点”时可进入，否则保持 |
| **工作站(3)** | 仅当“有当前任务 + 目标格为当前任务送货点”时可进入，否则保持 |
| **充电站(4)** | 仅当目标格为**本机编号对应充电站**时可进入（见下条），否则保持 |
| **空地(0)** | 可进入 |

---

## 四、充电站分配

| 规则 | 说明 |
|------|------|
| **编号对应** | 机器人 `robot_id` 使用的充电站下标 = `robot_id % len(charging_stations)`，即每机固定对应一个充电站 |

---

## 五、预测碰撞与礼让（同一步内的新位置冲突）

在计算完所有机器人的“新位置”后，先做**预测碰撞解析**（`_resolve_predictive_collisions`），再执行实际移动。冲突分两类：**同目标**（两人要去同一格）、**对穿**（两人互换位置）。处理顺序一致，如下。

| 优先级 | 情形 | 行为 |
|--------|------|------|
| 1 | **取货 vs 送货**（同目标或对穿） | 送货机器人：本步保持原位（等待）；取货机器人：左/右让一步，若两侧都不可走则后退一步 |
| 2 | **去充电 vs 取/送货**（同目标或对穿） | 去充电的机器人：本步保持原位；取/送货机器人：左/右让一步，否则后退一步 |
| 3 | **有任务 vs 无任务**（同目标或对穿） | 有任务的保持原计划移动；无任务的：左/右让一步，否则后退一步 |
| 4 | **其他冲突** | 由 `_choose_yielder` 选一方礼让：有任务的不动，没任务的让；若都有或都无则 **ID 大的** 礼让；礼让方本步保持原位 |

- **让路格**：左/右优先，不可走则沿“意图方向的反向”后退一步；若仍无合法格则保持原位。

---

## 六、执行移动时的现场礼让（目标被占）

| 规则 | 说明 |
|------|------|
| **充电中不让路** | 若目标格被“正在充电站充电”的机器人占用，当前机器人本步不移动（让路给充电的） |
| **按 ID 顺序移动** | 机器人按 ID 从小到大依次尝试移动，先移动的先占用新格 |
| **目标未被占** | 直接移动到新位置，并更新占用表 |
| **目标被占且本机挡路** | 若本机**无任务**且挡了别人：尝试移动到相邻空位（非 1、2，且未被占），随机顺序试四方向；有任务或充电中则不主动让路 |
| **目标被占且在工作站且已空闲** | 若本机在工作站且无任务/已完成任务，尝试移到相邻空位（非 1、2，且未被占） |
| **否则** | 保持原位置 |

---

## 七、电量与充电状态

| 规则 | 说明 |
|------|------|
| **耗电** | 非 CHARGING 状态每步按 `battery_consumption_rate` 扣电 |
| **需要充电** | `battery / max_battery < 0.2`（needs_charging）且不在 CHARGING/RETURNING_TO_CHARGE 时，可进入“去充电”逻辑 |
| **充电站判定** | 仅当机器人在**本机编号对应充电站**且电量 < 90% 时置为 CHARGING（可拿着货充电） |
| **充完电** | 电量 ≥ 90% 时：若有 `state_before_charge` 则恢复该状态，否则置 IDLE；若有 current_task 则 cancel_task |
| **去充电** | 需要充电且不在充电站时置为 RETURNING_TO_CHARGE；若有任务则先保存 state_before_charge |

---

## 八、任务状态机

| 状态 | 条件 / 转移 |
|------|-------------|
| MOVING_TO_PICKUP | 到达取货点 → PICKING_UP |
| PICKING_UP | 一步后 → MOVING_TO_DROPOFF；若当前在货架(2)上，自动移到相邻可通行格（不占其他机器人） |
| MOVING_TO_DROPOFF | 到达送货点(工作站) → DROPPING_OFF |
| DROPPING_OFF | 一步后 complete_task；并设置 `last_dropoff_workstation = 该工作站`，用于“离开工作站”规则 |

---

## 九、交付后离开工作站

| 规则 | 说明 |
|------|------|
| **触发** | 机器人执行完 DROPPING_OFF（complete_task）时，设置 `last_dropoff_workstation = 该工作站坐标` |
| **“所有工作站整体”** | 距离按**到任意一个工作站的最小曼哈顿距离**计算；须离开到与**所有工作站**都至少 3 格。 |
| **清除条件** | 每步 _update_robots 末尾：若机器人到所有工作站的最小距离 ≥ 3，则清空 `last_dropoff_workstation`；若无工作站列表则直接清空 |
| **常量** | `LEAVE_WORKSTATION_MIN_DISTANCE = 3` |

---

## 十、寻路（pathfinding）

| 规则 | 说明 |
|------|------|
| **可通行** | 在边界内、不在 occupied 中、非障碍(1)非货架(2)；若格为 3 或 4 则**仅当该格为 goal 时可通行**（不能穿过其他工作站/充电站） |
| **取货点** | `allow_goal_on_shelf=True` 时，goal 在货架上也可作为终点 |
| **path_to_action** | 路径前两步的位移 → 0=上, 1=右, 2=下, 3=左；路径长度<2 或无位移 → 4=等待 |

---

## 十一、启发式策略中的动作优先级（evaluate / inference）

用于无模型或混合时的规则策略，**从上到下**优先匹配：

| 优先级 | 条件 | 行为 |
|--------|------|------|
| 1 | 在充电站且电量 ≥ 90% | 尝试向四方向离开充电站（格非 1、2 且未被占则选该方向） |
| 2 | CHARGING 且电量 < 90% | 等待(4) |
| 3 | RETURNING_TO_CHARGE | A* 到本机对应充电站，path_to_action；若已在站则等待 |
| 4 | 有任务 | 取货阶段 A* 到 pickup_location（allow_goal_on_shelf=True），送货阶段 A* 到 dropoff_location（False），path_to_action |
| 5 | 无任务且需要充电 | A* 到本机对应充电站，path_to_action；已在站则等待 |
| 6 | 有 last_dropoff_workstation 且到所有工作站最小距离 < 3 | 从交付站沿四向试 d=3,4,5,… 的候选格；候选格须与**所有工作站**都至少 3 格且未被占；A* 可达则 path_to_action；若 3 格被占则试 4 格，以此类推 |
| 7 | 无任务且在工作站或充电站 | 尝试向四方向移动到相邻非 1、2 的格（inference 中还会试 2 步等） |
| 8 | 其他 | 等待(4) |

---

## 十二、Episode 终止与奖励

| 规则 | 说明 |
|------|------|
| **截断** | 步数达到 max_steps_per_episode 则 truncated |
| **终止** | 由环境定义的 terminated 条件（若存在） |
| **未完成任务惩罚** | 若在 episode 结束步仍有未完成任务，对所有机器人施加 episode_incomplete_penalty 的均摊负奖励 |

---

## 十三、任务生成（_spawn_tasks）

| 规则 | 说明 |
|------|------|
| **概率** | 每步以 `task_spawn_rate` 概率尝试生成一个新任务 |
| **上限** | 当前任务数 < max_tasks 才生成 |
| **取货点** | 在货架区域内的格中随机选一个 |
| **送货点** | 在 workstation_positions 中随机选一个 |
| **任务类型** | PICKUP，含 pickup_location、dropoff_location、priority、created_time 等 |

---

以上为当前仓库系统中与逻辑、规则相关的全部条目；实现细节以代码为准，本文档仅作索引与核对用。
